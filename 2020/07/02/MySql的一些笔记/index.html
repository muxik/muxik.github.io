开始

​ 为啥重新学习？在过去很长一段时间里,我做 PHP 开发都是使用框架，这些框架在进行数据操作时非常方便，甚至不用自己写 SQL 语句，这使我忘了很多之前学习的 MySQL 数据库知识。

​ 最近接到学校通知，开学有MySQL考试，所以我决定重学MySQL 避免一些不必要的麻烦

为什么要学习数据库？

  1. 实现数据持久化
  2. 数据结构化，易于查询和管理
  3. 数据的共享性高，冗余度低，易于扩充
  4. 由DBMS统一管理和控制，易于维护

数据库相关概念

1. DB

​ 数据库( database ) : 储存数据的”仓库” ，它保存了一系列有组织的数据

2. DBMS

​ 数据库管理系统( Database Management System ) 。数据库是通过DBMS创建和操作的容器

3. SQL

​ 机构化查询语言( Structure Query Language ) : 专门用来与数据库通信的语言

  • SQL的优点：
    • 不是某个特点的数据库供应商专有的语言，几乎所有 DBMS 都支持SQL
    • 简单易学
    • 虽然简单，但实际上是一种强有力的语言，灵活使用语言元素，可以进行非常复杂和高级的数据库操作

#　MySQL存储数据过程

概念

  • 表头(header): 每一列的名称;
  • 列(col): 具有相同数据类型的数据的集合;
  • 行(row): 每一行用来描述某条记录的具体信息;
  • 值(value): 行的具体信息, 每个值必须与该列的数据类型相同;
  • 键(key): 键的值在当前列中具有唯一性。

过程

  1. 将数据放到表中，表在放到库中
  2. 一个数据库可以有多个表，每个表都有一个名字，用来标识自己，表名具有唯一性。
  3. 表具有一些特性，这些特性定义了数据在表中如何存储，类似 面向对象 中类的设计
  4. 表又列组成，我们也称为字段。 所有表都是由一个或者多个列组成，类似于面向对象中类的熟悉
  5. 表中的数据是按行存储的，每一行类似与面向对象中的对象

#　MySQL介绍

​ MySQL 是最流行的关系型数据库管理系统，在 WEB 应用方面 MySQL 是最好的 RDBMS(Relational Database Management System：关系数据库管理系统)应用软件之一。

MySQL 是一个关系型数据库管理系统，由瑞典 MySQL AB 公司开发，目前属于 Oracle 公司。MySQL 是一种关联数据库管理系统，关联数据库将数据保存在不同的表中，而不是将所有数据放在一个大仓库内，这样就增加了速度并提高了灵活性。

  • MySQL 是开源的，所以你不需要支付额外的费用。
  • MySQL 支持大型的数据库。可以处理拥有上千万条记录的大型数据库。
  • MySQL 使用标准的 SQL 数据语言形式。
  • MySQL 可以运行于多个系统上，并且支持多种语言。这些编程语言包括 C、C++、Python、Java、Perl、PHP、Eiffel、Ruby 和 Tcl 等。
  • MySQL 对PHP有很好的支持，PHP 是目前最流行的 Web 开发语言。
  • MySQL 支持大型数据库，支持 5000 万条记录的数据仓库，32 位系统表文件最大可支持 4GB，64 位系统支持最大的表文件为8TB。
  • MySQL 是可以定制的，采用了 GPL 协议，你可以修改源码来开发自己的 MySQL 系统。

你可以在/etc/mysql/my.cnf路径下找到这个文件并进行一些配置

$ sudo systemctl restart mysqld.service

DQL条件查询

简介

  • 语法

    select
    查询列表
from 
    表名
where
    筛选条件

  • 分类

    • 按条件表达式筛选

      • 条件运算符: > < = != <> >= <=

    • 按逻辑表达式筛选

      • 逻辑运算符:

        • && and两个条件都为 true 则结果为 true 反之 false
        • || or只要有一个条件为 true 则结果为 true 反之 false
        • ! not如果连接的条件为 true 则结果为 false 反之 true

    • 模糊查询

      • like
      • between and
      • in
      • is null

数据库相关概念

###　一、数据库好处

  1. 持久化数据到本地
  2. 结构化查询

二、数据库的常见概念

  1. DB：数据库
  2. DBMS：数据库管理系统，又称为数据库软件或数据库产品，用于创建或管理DB
  3. SQL：结构化查询语言

三、数据库存储数据的特点

  1. 数据存放到表中，然后表被放到库中
  2. 一个库中可以有多张表， 每张表具有唯一的表名用来标识自己
  3. 表中的有一个或者多个列，又称为字段
  4. 表中的每一行数据，相当于对象

四、常见的数据库管理系统

  1. MySQL
  2. Oracle
  3. db2
  4. sqlserver

##　ＭySQL 的介绍

一、MySQL的背景

1. 前身属于瑞典的一家公司，MySQL AB

2. 2008年被sun公司收购

3. 2009年sun被oracle收购

二、MySQL的优点

  1. 开源，免费，成本低
  2. 性能高，移植性好
  3. 体积小，便于安装

三、MySQL服务启动停止

  • Linux(manjaro) 

    $ systemctl start mysqld # 启动
$ systemctl stop mysqld # 停止

  • Windows(Administrator)

    net start mysqld :: 启动
net stop mysqld  :: 停止

四、MySQL服务登录退出

$ mysql [-h主机名 -P端口号] -u 用户名 -p密码

DQL

基础查询

  • 语法:

    select 查询列表　from 表名;

  • 特点:

    1. 查询列表可以是字段、常量、表达式、函数、也可以是多个
    2. 查询结果是一个虚拟的表

  • 实例

    1. 查询单个字段

      select 字段名 from 表名;

    2. 查询多个字段

      select 字段名，字段名 from 表名;

    3. 查询多个字段

      select * from 表名;

    4. 查询常量 

      select 常量值; -- 字符型|日期型常量值要用引号引起来，数值型不需要

    5. 查询函数

      select 函数名(实参列表);

    6. 查询表达式

    select 1+1;

    1. 起别名

      1. AS

        select 字段名 as 别名 from 表名;

      2. 空格

        select 字段名 别名 from 表名;

  1. 去重

    select distinct 字段名 from 表名;

  2. +

    作用：加法运算

    select  1+1; -- succees 2
select '1'1; -- succees 2
select 'a'+1; -- error 1
select null + 1 -- error null

  3. [补充] concat()

    功能：拼接字符串

    select concat('str1','str2');

  4. [补充] ifnull()

    功能：判断某个字段的值是否为null 返回指定值，否则返回原本值

    select ifnull(null,0)

  5. [补充] ifnull()

    功能：判断某个字段的值是否为null 返回bool值

    select isnull(null)

条件查询

  • 语法:

    select 字段名 from 表名 where 条件;

  • 分类

    1. 简单条件运算符

      >,<,=,!=,<>, <=>,>=,<=

    2. 逻辑运算符

      && and,|| or,! not

    3. 模糊查询

      like, between and ,in

常见函数

  • 概念: 将一段逻辑代码封装，对外暴露函数名

  • 好处：1. 隐藏实现细节，2.提高代码复用性

  • 语法： select 函数名（实参）

  • 特点：

    1. 叫什么(函数名)
    2. 干什么(函数功能)

  • 分类：

    1. 单行函数

      如: concat(), length(), ifnull()

    2. 分组函数

      功能：做统计使用，又称为统计函数、聚合函数、组函数

  • 学习目标：

    • 字符函数
    • 数学函数
    • 日期函数
    • 其他函数
    • 流程控制函数
